## 一、项目概述

### 1.1 项目定位

AI虚拟教师系统是一款面向K12学生的个性化智能教学产品。系统通过预设的课程知识图谱，结合大语言模型的讲解与交互能力，为学生提供自适应的学习路径和精准的知识点讲解。

### 1.2 MVP核心验证目标

| 验证目标 | 验证方式 | 成功标准 |
|---------|---------|---------|
| AI讲解+提问的效果 | 学生完成知识点后的测试正确率 | 平均正确率 ≥ 70% |
| 回溯补救的有效性 | 回溯后学生掌握率提升幅度 | 提升幅度 ≥ 30% |
| 学习目标达成率 | 完成单元学习后的综合测评 | 达标率 ≥ 80% |

### 1.3 MVP试点范围

- **学科**: 数学
- **单元**: 初一数学"一次函数"
- **知识点数量**: 32个
- **目标用户**: 初一学生

## 二、知识点图谱（一次函数单元）

### 2.1 知识点列表

#### 第一层：基础概念层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K1 | 坐标系 | 概念 | 平面直角坐标系的建立，x轴、y轴、原点 |
| K2 | 点的坐标 | 技能 | 用有序数对(x,y)表示平面上点的位置 |
| K3 | 象限 | 概念 | 坐标平面被两轴分成四个象限及其特征 |
| K4 | 变量 | 概念 | 可以取不同数值的量 |
| K5 | 常量 | 概念 | 数值固定不变的量 |

#### 第二层：函数基础层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K6 | 函数定义 | 概念 | 设x和y是两个变量，若x每取一个值，y都有唯一确定的值与之对应 |
| K7 | 自变量与因变量 | 概念 | 主动变化的量为自变量，随之变化的量为因变量 |
| K8 | 函数的定义域 | 概念 | 自变量x允许取值的范围 |
| K9 | 函数值 | 技能 | 当x取某值时，y对应的值 |
| K10 | 函数解析式 | 概念 | 用数学式子表示函数关系 |
| K11 | 函数的三种表示法 | 技能 | 解析式法、列表法、图象法 |

#### 第三层：一次函数核心层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K12 | 正比例函数定义 | 概念 | y=kx(k≠0)形式的函数 |
| K13 | 正比例函数图象 | 技能 | 正比例函数图象是过原点的直线 |
| K14 | 正比例函数性质 | 公式 | k>0过一三象限且递增；k<0过二四象限且递减 |
| K15 | 一次函数定义 | 概念 | y=kx+b(k≠0)形式的函数 |
| K16 | 一次函数与正比例函数关系 | 概念 | 当b=0时，一次函数退化为正比例函数 |

#### 第四层：图象与性质层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K17 | 描点法画函数图象 | 技能 | 列表、描点、连线的作图方法 |
| K18 | 一次函数图象特征 | 技能 | 一次函数图象是一条直线 |
| K19 | 两点确定直线 | 技能 | 画一次函数图象只需描出两点连线 |
| K20 | 截距概念 | 概念 | 直线与y轴交点的纵坐标，即b的值 |
| K21 | 斜率概念 | 概念 | 直线倾斜程度的度量，即k的值 |
| K22 | k对图象方向的影响 | 公式 | k>0从左向右上升；k<0从左向右下降 |
| K23 | b对图象位置的影响 | 公式 | b>0与y轴交于正半轴；b<0交于负半轴 |
| K24 | 一次函数增减性 | 公式 | k>0时y随x增大而增大；k<0时y随x增大而减小 |

#### 第五层：图象变换层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K25 | 平移变换 | 技能 | 上下平移改变b值，左右平移改变x |

#### 第六层：综合判断层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K26 | k与b的综合判断 | 技能 | 根据k、b符号判断图象经过的象限 |

#### 第七层：应用层

| 编号 | 名称 | 类型 | 简述 |
|------|------|------|------|
| K27 | 待定系数法 | 技能 | 设出函数解析式，代入条件求k、b |
| K28 | 由图象求解析式 | 技能 | 从图象上读取点坐标，用待定系数法求解析式 |
| K29 | 求交点坐标 | 技能 | 联立两一次函数解析式求解方程组 |
| K30 | 实际问题建模 | 技能 | 将实际问题转化为一次函数求解 |
| K31 | 一次函数与方程的关系 | 概念 | 求ax+b=0的解等价于求y=ax+b与x轴交点横坐标 |
| K32 | 一次函数与不等式的关系 | 概念 | 图象在x轴上方/下方对应的x范围即为不等式解集 |

### 3.2 各模块详细说明

#### 3.2.1 课程图谱引擎

**职责**: 管理、存储、查询课程知识图谱

**核心数据结构**:

```json
{
  "knowledge_point": {
    "id": "K15",
    "name": "一次函数定义",
    "type": "概念",
    "description": "y=kx+b(k≠0)形式的函数",
    "level": 3,
    "dependencies": ["K10", "K12"],
    "mastery_criteria": {
      "type": "concept_check",
      "method": "选择题",
      "question_count": 2,
      "pass_threshold": 1
    },
    "teaching_config": {
      "use_examples": true,
      "ask_questions": true,
      "question_positions": ["讲解中段", "讲解结束"]
    }
  }
}
```

#### 3.2.2 学习路径管理器

**职责**: 管理学生的学习进度，决定下一个要学习的知识点

**核心逻辑**:
- 初始化：从Level 0开始，按层级顺序推进
- 前置检查：学习某知识点前，确保其所有直接前置知识点已掌握
- 进度保存：每次状态变更后持久化

**状态流转**:

```
待学习 → 学习中 → 待验证 → 已掌握 / 需回溯
                ↑_______________|
```

**学生主动权设计**:

| 主动行为 | 支持情况 | 系统处理方式 | 档案记录 |
|---------|---------|-------------|---------|
| 主动跳过知识点 | 支持 | 标记为“已跳过”，后续可被回溯机制重新触发 | 记录跳过时间、知识点ID、跳过原因(可选) |
| 主动要求复习 | 支持 | 立即切换到指定知识点，使用复习版讲解 | 记录复习请求时间、目标知识点 |
| 主动提问 | 支持 | AI实时响应，不改变当前学习进度 | 记录在会话日志中 |

**跳过知识点的风险控制**:
- 跳过的知识点在后续学习中被识别为前置依赖时，系统会提示学生先完成
- 学习档案中明确标记“跳过”状态，供老师/家长查看
- 单元结束时，对跳过的知识点进行抽查测试

#### 3.2.3 知识点讲解器

**职责**: 调用大模型生成知识点讲解内容

**讲解模板配置**:

```json
{
  "teaching_template": {
    "knowledge_point_id": "K15",
    "structure": [
      {"section": "引入", "content": "从生活实例引入"},
      {"section": "定义", "content": "给出正式定义"},
      {"section": "示例", "content": "举例说明", "optional": true},
      {"section": "提问", "content": "检查理解", "optional": true},
      {"section": "总结", "content": "归纳要点"}
    ]
  }
}
```

**复习版讲解机制**:

当学生回溯或主动复习某知识点时，系统采用“复习版讲解”策略，避免内容重复导致的厌倦感。

| 讲解版本 | 触发条件 | Prompt设计要点 |
|---------|---------|---------------|
| 首次讲解 | 学生首次学习该知识点 | 完整讲解，包含引入、定义、示例、提问、总结 |
| 复习版讲解v1 | 学生第2次学习该知识点 | "这是学生第二次学习这个知识点，请用不同的方式讲解，重点放在学生上次出错的地方" |
| 复习版讲解v2 | 学生第3次学习该知识点 | "这是学生第三次学习这个知识点，请用更简洁的方式总结要点，并增加练习题巩固" |

**Prompt模板示例**:

```
你是一位初中数学老师，正在辅导学生学习《一次函数》单元。

当前知识点：{knowledge_point_name}
知识点描述：{description}

学生情况：
- 这是学生第{attempt_count}次学习这个知识点
- 上次学习时间：{last_learning_time}
- 上次评估结果：{last_assessment_result}
- 学生主要错误：{error_analysis}

讲解要求：
{teaching_requirements}

请用亲切、耐心的语气进行讲解，每个讲解段落控制在100字以内。
```

**teaching_requirements 动态生成逻辑**:

```python
if attempt_count == 1:
    requirements = "完整讲解概念，包含定义、例子和练习"
elif attempt_count == 2:
    requirements = "使用不同的讲解方式，避免与上次重复，重点关注学生上次出错的地方"
else:
    requirements = "简洁总结要点，增加练习题巩固，帮助学生建立信心"
```

#### 3.2.4 交互验证模块

**职责**: 在关键点主动提问，验证学生理解程度

**交互类型**:

| 交互类型 | 适用知识类型 | 示例 |
|---------|-------------|------|
| 概念确认 | 概念 | "你能用自己的话说说什么是函数吗？" |
| 选择题 | 公式/概念 | "下面哪个是一次函数？A. y=2x B. y=x² C. y=√x" |
| 填空题 | 公式 | "一次函数的一般形式是 y=___+___" |
| 判断题 | 性质 | "k>0时，y随x增大而减小，对吗？" |

#### 3.2.5 回溯决策器

**职责**: 当学生未掌握某知识点时，决定回溯到哪个前置知识点

**决策机制**: 由大模型实时推断

**决策输入**:
- 当前知识点ID
- 学生错误表现（错误答案、错误类型）
- 前置知识点列表
- 学生在各前置知识点的历史表现

**决策输出**:
```json
{
  "action": "backtrack",
  "target_knowledge_point": "K12",
  "reason": "学生对一次函数定义理解有误，可能是正比例函数定义掌握不牢"
}
```

#### 3.2.6 掌握度评估模块

**职责**: 判断学生是否掌握某知识点

**评估标准（按知识点类型）**:

| 知识类型 | 评估方式 | 题目数量 | 通过阈值 | 可调整 |
|---------|---------|---------|---------|--------|
| 概念 | 选择题/判断题 | 2题 | 答对1题 | 是 |
| 公式 | 填空题/选择题 | 3题 | 答对2题 | 是 |
| 技能 | 计算题/作图题 | 2题 | 答对1题 | 是 |

**阈值动态调整**: MVP阶段使用固定阈值，后续版本基于学生群体数据统计自动优化阈值。调整机制待后续版本设计。

## 十、风险与待决策项

### 10.1 待决策项

| 编号 | 问题 | 影响范围 | 建议决策时间 |
|------|------|---------|-------------|
| D1 | 是否支持学生主动跳过某知识点？ | 学习路径 | **已决策：支持，档案中记录** |
| D2 | 是否支持学生主动要求复习？ | 学习路径 | **已决策：支持，作为回溯补充** |
| D3 | 白板是否支持手写输入？ | 用户体验 | Phase 4 |
| D4 | 是否支持学生查看自己的学习地图？ | 用户体验 | Phase 3 |
| D5 | 遗忘曲线和间隔复习机制何时引入？ | 产品规划 | MVP后 |

### 10.2 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| LLM响应延迟 | 用户体验差 | 流式输出 + 缓存常见讲解 |
| LLM偏离预设路径 | 学习效果不可控 | 强约束prompt + 路径校验 |
| 语音识别准确率 | 学生输入理解错误 | 提供文字输入备选方案 |
| 回溯决策不稳定 | 学习路径混乱 | 增加规则兜底 + 人工审核日志 |

### 10.3 业务风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 学生对AI讲解不适应 | 用户流失 | 提供讲解风格偏好设置 |
| 回溯过多导致学习时长过长 | 用户流失 | 设置回溯上限（后续版本） |
| 课程图谱设计不合理 | 学习效果差 | 专家审核 + 迭代优化 |
