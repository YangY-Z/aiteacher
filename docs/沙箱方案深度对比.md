## 一、核心问题：Docker是否太重？

### 1.1 Docker的"重"体现在哪里？

| 维度 | Docker的开销 | 实际影响 |
|------|-------------|---------|
| **启动时间** | 0.5-2秒 | 对于交互式教学，可能影响用户体验 |
| **内存占用** | 每个容器 ~10-50MB | 并发10个容器 = 500MB基础开销 |
| **CPU开销** | 创建容器时短暂峰值 | 可接受 |
| **磁盘IO** | 镜像加载、文件系统操作 | 影响较小 |
| **并发限制** | 受限于宿主机资源 | 高并发场景可能成为瓶颈 |

### 1.2 在Manim场景下的实际影响

**场景分析：**
```
学生请求: "展示一次函数 y=2x+1 的图像"
↓
流程:
1. Agent决策需要动画 (<100ms)
2. LLM生成Manim代码 (~1-2s)
3. 启动Docker容器 (~1s)          ← 这里有开销
4. 渲染动画 (10-30s)              ← 主要耗时
5. 返回视频URL (<100ms)

总耗时: 12-35秒
Docker占比: ~3-8%
```

**结论：**
- ✅ **Docker开销可接受** - 主要耗时在渲染，Docker占比很小
- ✅ **安全性更重要** - 动画生成是LLM生成代码，安全隔离不可妥协
- ⚠️ **高并发场景需优化** - 如果同时有100+学生，容器管理需要优化

## 四、Firecracker 微虚拟机方案

### 4.1 为什么Firecracker更轻量？

**技术原理对比：**

```
Docker容器:
┌──────────────────┐
│  App Process     │
├──────────────────┤
│  Container Engine│
├──────────────────┤
│  Shared Kernel   │ ← 安全风险：共享内核
├──────────────────┤
│  Host OS         │
└──────────────────┘

Firecracker microVM:
┌──────────────────┐
│  App Process     │
├──────────────────┤
│  Guest Kernel    │ ← 硬件隔离：独立内核
├──────────────────┤
│  Firecracker VMM │
├──────────────────┤
│  Host Kernel     │
└──────────────────┘
```

**性能对比：**

| 指标 | Docker | Firecracker | 提升 |
|------|--------|-------------|------|
| 启动时间 | 0.5-2s | **0.1-0.5s** | **2-4倍** |
| 内存占用 | 10-50MB | **5-10MB** | **50%↓** |
| 安全隔离 | 容器级 | **硬件级** | **更强** |
| 密度（单机） | ~100容器 | **~1000 microVM** | **10倍** |

### 4.2 Firecracker 的挑战

| 挑战 | 说明 | 解决方案 |
|------|------|---------|
| **配置复杂** | 需要准备rootfs、kernel | OpenSandbox已封装 |
| **生态不如Docker** | 工具链较少 | 通过OpenSandbox使用 |
| **调试困难** | 无法exec进入容器 | 通过OpenSandbox API操作 |

**推荐方案：通过OpenSandbox使用Firecracker**

```python
# 只需配置文件切换运行时
# ~/.sandbox.toml

[runtime]
type = "firecracker"  # 或 "docker"

[firecracker]
kernel_path = "/path/to/vmlinux"
rootfs_path = "/path/to/rootfs.ext4"
```

```python
# ai-teacher-backend/app/services/animation_generator_v2.py

import asyncio
from pathlib import Path
from typing import Dict, Any, Optional
from opensandbox import Sandbox
from code_interpreter import CodeInterpreter, SupportedLanguage

from app.services.llm_service import llm_service
import logging

logger = logging.getLogger(__name__)


class OpenSandboxAnimationGenerator:
    """基于OpenSandbox的动画生成器"""
    
    def __init__(
        self,
        sandbox_image: str = "opensandbox/code-interpreter:latest",
        default_timeout: int = 60,
    ):
        self.sandbox_image = sandbox_image
        self.default_timeout = default_timeout
        logger.info(f"OpenSandboxAnimationGenerator 初始化: image={sandbox_image}")
    
    async def generate_animation(
        self,
        manim_code: str,
        animation_id: str,
        trace_id: Optional[str] = None,
    ) -> Dict[str, Any]:
        """使用OpenSandbox执行Manim代码
        
        Args:
            manim_code: Manim Python代码
            animation_id: 动画ID（用于缓存）
            trace_id: 追踪ID
            
        Returns:
            {
                "video_url": "/media/xxx.mp4",
                "file_path": "/path/to/video.mp4",
                "duration": 15.0,
            }
        """
        trace_id = trace_id or animation_id
        logger.info(f"[{trace_id}] 开始在OpenSandbox中执行Manim代码")
        
        # Step 1: 创建沙箱
        sandbox = await Sandbox.create(
            self.sandbox_image,
            env={
                "MANIM_QUALITY": "medium_quality",
                "PYTHON_VERSION": "3.11",
            },
            timeout=self.default_timeout,
        )
        
        try:
            async with sandbox:
                # Step 2: 写入Manim代码
                await sandbox.files.write_files([
                    {
                        "path": "/workspace/animation.py",
                        "data": manim_code,
                        "mode": 0o644,
                    }
                ])
                
                logger.info(f"[{trace_id}] 代码已写入沙箱")
                
                # Step 3: 执行Manim渲染
                execution = await sandbox.commands.run(
                    "cd /workspace && manim -qh animation.py AnimationScene"
                )
                
                if execution.exit_code != 0:
                    error_msg = execution.logs.stderr[0].text if execution.logs.stderr else "Unknown error"
                    logger.error(f"[{trace_id}] Manim执行失败: {error_msg}")
                    raise RuntimeError(f"Manim执行失败: {error_msg}")
                
                logger.info(f"[{trace_id}] Manim渲染完成")
                
                # Step 4: 查找生成的视频文件
                video_files = await sandbox.files.list_dir("/workspace/media/videos/animation")
                mp4_files = [f for f in video_files if f.endswith(".mp4")]
                
                if not mp4_files:
                    raise FileNotFoundError("未找到生成的视频文件")
                
                video_path_in_sandbox = mp4_files[0]
                
                # Step 5: 读取视频文件
                video_data = await sandbox.files.read_file(video_path_in_sandbox)
                
                # Step 6: 保存到本地
                local_video_path = Path(f"./generated_media/{animation_id}.mp4")
                local_video_path.parent.mkdir(parents=True, exist_ok=True)
                local_video_path.write_bytes(video_data)
                
                logger.info(f"[{trace_id}] 视频已保存: {local_video_path}")
                
                return {
                    "video_url": f"/media/{animation_id}.mp4",
                    "file_path": str(local_video_path),
                    "duration": self._get_video_duration(str(local_video_path)),
                }
                
        finally:
            # 清理沙箱
            await sandbox.kill()
    
    def _get_video_duration(self, video_path: str) -> float:
        """获取视频时长（简化版）"""
        # 实际应使用ffprobe
        return 15.0  # 临时返回


# 全局实例
animation_generator_v2 = OpenSandboxAnimationGenerator()
```
