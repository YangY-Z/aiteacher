# 章节数据迁移和前端集成总结

## 一、后端变更

### 1. Chapter模型更新

**文件**: `ai-teacher-backend/app/models/course.py`

**变更内容**:
```python
@dataclass
class Chapter:
    id: str
    name: string
    grade: str  # 保留用于兼容
    edition: Edition
    subject: Subject  # 保留用于兼容
    grade_id: Optional[str] = None  # 新增：年级ID外键
    subject_id: Optional[str] = None  # 新增：科目ID外键
    # ... 其他字段
```

**说明**:
- 添加 `grade_id` 和 `subject_id` 字段作为外键引用
- 保留原有 `grade` 和 `subject` 字段用于向后兼容
- 渐进式迁移，不影响现有功能

### 2. ChapterRepository创建

**文件**: `ai-teacher-backend/app/repositories/chapter_repository.py`

**功能**:
- 章节数据持久化（JSON文件）
- 从内存数据库加载数据
- 按年级ID、科目ID查询章节
- 数据迁移功能

**关键方法**:
```python
def migrate_grade_subject_references(
    self, grade_mapping: dict[str, str], subject_mapping: dict[str, str]
) -> int:
    """迁移grade和subject引用到外键ID"""
```

### 3. 数据迁移脚本

**文件**: `scripts/migrate_chapter_references.py`

**执行结果**:
```
✅ 迁移完成！更新了 2 个引用
✓ 一次函数: grade_id=G_C2, subject_id=S_MATH
总计：1 个章节，1 个已迁移
```

**迁移逻辑**:
1. 从 grade_repository 加载年级映射: `{name: id}`
2. 从 subject_repository 加载科目映射: `{name: id}`
3. 遍历章节，将 grade 和 subject 字段映射到 grade_id 和 subject_id
4. 保存到 `data/chapters.json`

### 4. 数据文件

**新增文件**:
- `data/chapters.json` - 章节数据持久化

##### loadConfig() - 加载配置
```typescript
// 旧代码（localStorage）
const loadConfig = () => {
  const savedGrades = localStorage.getItem('admin_grades');
  if (savedGrades) {
    setGrades(JSON.parse(savedGrades));
  }
  // ...
};

// 新代码（API）
const loadConfig = async () => {
  const [gradesRes, subjectsRes] = await Promise.all([
    gradeApi.list({ active_only: true }),
    subjectApi.list({ active_only: true })
  ]);
  setGrades(gradesRes.grades);
  setSubjects(subjectsRes.subjects);
};
```

const handleAddGrade = async (name: string) => {

## 三、数据流程对比

### 旧流程（localStorage）
```
前端 localStorage
    ↓
AdminCourses 组件
    ↓
loadConfig() - 从 localStorage 读取
    ↓
handleAddItem() - 更新 localStorage
    ↓
organizeBooks() - 组织展示
```

### 新流程（API + 数据库）
```
前端组件
    ↓
gradeApi / subjectApi
    ↓
FastAPI Endpoints (/api/v1/admin/grades, /api/v1/admin/subjects)
    ↓
GradeService / SubjectService
    ↓
GradeRepository / SubjectRepository
    ↓
JSON文件持久化 (data/grades.json, data/subjects.json)
```

### 年级管理
