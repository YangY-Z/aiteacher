### 3. 递归前置依赖显示

**修改文件:**
- `src/components/KnowledgeGraph.tsx` - 递归算法实现

**核心算法:**
```typescript
// 递归获取所有前置依赖节点ID
const getAllPrerequisites = (nodeId: string, links): Set<string> => {
  const prerequisites = new Set<string>();
  
  const findPrerequisites = (id: string) => {
    // 找到直接前置依赖
    const directPrereqs = links
      .filter(link => link.target === id)
      .map(link => link.source);
    
    directPrereqs.forEach(prereqId => {
      if (!prerequisites.has(prereqId)) {
        prerequisites.add(prereqId);
        // 递归查找前置依赖的前置依赖
        findPrerequisites(prereqId);
      }
    });
  };
  
  findPrerequisites(nodeId);
  return prerequisites;
};
```

**实现逻辑:**
1. 悬浮节点时,调用递归函数获取所有前置依赖
2. 显示所有依赖链中的连线
3. 高亮所有前置依赖节点,其他节点淡化

**示例:**
悬浮 "一次函数定义"(K15) 时显示:
- K15 ← K12 (正比例函数定义)
- K15 ← K10 (函数解析式)
- K12 ← K6, K10 (递归)
- K6 ← K4, K5 (继续递归)

## 技术要点

### D3.js 布局选择

**问题:** 原有力导向图节点位置不稳定,看起来很乱

**解决方案:** 
- 放弃力导向模拟 (`d3.forceSimulation`)
- 改用固定位置计算
- 手动控制节点坐标

**优势:**
- 布局稳定,不会移动
- 层级清晰
- 更好的可读性

### 递归算法注意事项

**性能优化:**
- 使用 `Set` 去重,避免重复访问
- 记忆化已访问节点
- 避免无限递归(依赖循环)

**边界情况:**
- 无依赖节点返回空 Set
- 自引用依赖被 Set 自动去重
