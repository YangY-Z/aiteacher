## 详细内容

### 1. 知识图谱初始化失败

**问题描述:**
```
初始化知识图谱失败: ReferenceError: level is not defined
```

**错误原因:**
在 `levelInfos.forEach` 循环中错误使用了未定义的 `level` 变量

**修复方案:**
```typescript
// ❌ 错误写法
const levelDesc = levelDescriptions[level] || ...

// ✅ 正确写法  
const levelDesc = levelDescriptions[levelInfo.level] || ...
```

**影响范围:** 知识图谱完全无法渲染

**验证:** 重启后端后页面正常显示

### 2. 节点悬浮时抽搐

**问题描述:**
鼠标悬浮在知识点上时,节点会发生抖动/抽搐

**错误原因:**
CSS 的 `transform: scale(1.05)` 在 SVG transform 属性上冲突

**修复方案:**
```css
/* ❌ 移除 */
.node-group:hover {
  transform: scale(1.05);
}

/* ✅ 改用 */
.node-group:hover .node-rect {
  stroke-width: 3;
  filter: drop-shadow(0 4px 12px rgba(0, 0, 0, 0.15));
}
```

**效果:**
- 悬浮时边框加粗
- 阴影加深
- 无抖动,流畅自然

### 3. 连线显示逻辑错误

**问题描述:**
- 显示了后置依赖连线(当前节点 → 后续节点)
- 没有递归显示完整的依赖链

**原始逻辑:**
```typescript
// ❌ 显示源或目标的连线
if (sourceId === d.id || targetId === d.id) {
  return 1;
}
```

**修复方案:**
```typescript
// ✅ 只显示前置依赖
if (targetId === d.id || allPrereqIds.has(targetId)) {
  return 1;
}
```

**改进:**
- 只显示前置依赖(不显示后置)
- 递归显示完整依赖链
- 更符合学习路径逻辑

### 4. 层级描述不显示

**问题描述:**
层级框内只显示 "Level 1" 等通用标签,不显示具体描述

**错误原因:**
后端数据模型缺少 `level_descriptions` 字段

**修复方案:**

**后端数据模型:**
```python
@dataclass
class Course:
    # ... 其他字段
    level_descriptions: dict[int, str] = field(default_factory=dict)
```

**初始化数据:**
```python
course = Course(
    id="MATH_JUNIOR_01",
    # ...
    level_descriptions={
        0: "基础概念层",
        1: "核心概念层",
        # ... 7个层级
    }
)
```

**前端类型:**
```typescript
export interface Course {
  // ...
  level_descriptions?: Record<number, string>;
}
```

**验证:** 重启后端服务后生效

## 技术要点

### 变量作用域问题

**教训:**
- 在 `forEach` 回调中注意变量作用域
- 使用参数传递而非外部变量
- TypeScript 严格模式有助于提前发现

**最佳实践:**
```typescript
array.forEach((item, index) => {
  // ✅ 使用 item.property
  // ❌ 避免使用外部同名变量
});
```

### CSS Transform 冲突

**问题:**
SVG 元素的 `transform` 属性与 CSS `transform` 冲突

**解决方案:**
- SVG 元素避免使用 CSS transform
- 改用 SVG 属性或滤镜效果
- 使用 stroke, filter 等非 transform 属性

### 数据初始化顺序

**问题:**
后端数据在内存数据库初始化时加载

**注意:**
- 修改数据模型后需要重启服务
- 内存数据库的数据在启动时初始化
- 持久化数据需要特殊处理
