## 详细内容

### 1. 年级科目管理API开发

**修改文件:**
- `ai-teacher-backend/app/api/admin.py` - 年级科目管理API端点（418行）
- `ai-teacher-backend/app/services/grade_service.py` - 年级领域服务
- `ai-teacher-backend/app/services/subject_service.py` - 科目领域服务
- `ai-teacher-backend/app/repositories/grade_repository.py` - 年级仓储实现
- `ai-teacher-backend/app/repositories/subject_repository.py` - 科目仓储实现

**实现细节:**

#### 年级管理API端点
- `GET /api/v1/admin/grades` - 获取年级列表（支持按学段过滤）
- `POST /api/v1/admin/grades` - 创建年级
- `GET /api/v1/admin/grades/{id}` - 获取年级详情
- `PUT /api/v1/admin/grades/{id}` - 更新年级
- `DELETE /api/v1/admin/grades/{id}` - 删除年级（级联删除关联）

#### 科目管理API端点
- `GET /api/v1/admin/subjects` - 获取科目列表（支持按类别过滤）
- `POST /api/v1/admin/subjects` - 创建科目
- `GET /api/v1/admin/subjects/{id}` - 获取科目详情
- `PUT /api/v1/admin/subjects/{id}` - 更新科目
- `DELETE /api/v1/admin/subjects/{id}` - 删除科目

#### 年级-科目关联管理
- `GET /api/v1/admin/grades/{id}/subjects` - 获取年级下的科目
- `POST /api/v1/admin/grades/{id}/subjects` - 为年级添加科目
- `DELETE /api/v1/admin/grades/{id}/subjects/{subject_id}` - 从年级移除科目

**关键设计:**
- 使用FastAPI框架，支持异步操作
- 统一的错误处理机制
- 完整的类型注解和文档字符串
- RESTful规范设计

### 4. API接口封装

**修改文件:**
- `ai-teacher-frontend/src/api/admin.ts` - 前端API接口封装

**实现内容:**
```typescript
// 年级API
export const gradeApi = {
  list: (params?) => request.get('/admin/grades', { params }),
  create: (data) => request.post('/admin/grades', data),
  get: (id) => request.get(`/admin/grades/${id}`),
  update: (id, data) => request.put(`/admin/grades/${id}`, data),
  delete: (id) => request.delete(`/admin/grades/${id}`),
  getSubjects: (id) => request.get(`/admin/grades/${id}/subjects`),
  addSubject: (id, data) => request.post(`/admin/grades/${id}/subjects`, data),
  removeSubject: (id, subjectId) => request.delete(`/admin/grades/${id}/subjects/${subjectId}`),
}

// 科目API
export const subjectApi = {
  list: (params?) => request.get('/admin/subjects', { params }),
  create: (data) => request.post('/admin/subjects', data),
  get: (id) => request.get(`/admin/subjects/${id}`),
  update: (id, data) => request.put(`/admin/subjects/${id}`, data),
  delete: (id) => request.delete(`/admin/subjects/${id}`),
```

## 技术要点

### 1. 架构设计
- **分层架构**: API层 -> 服务层 -> 仓储层 -> 数据层
- **DDD设计**: 领域模型驱动，聚合边界清晰
- **RESTful API**: 符合REST规范，资源导向设计

### 2. 数据管理
- **JSON持久化**: 使用JSON文件存储，便于开发调试
- **外键引用**: grade_id和subject_id外键关系
- **级联操作**: 删除年级时级联删除关联关系

### 3. 前端集成
- **API封装**: 统一的API调用接口
- **类型安全**: TypeScript类型定义
- **状态管理**: React Hooks状态管理

### 4. 错误处理
- **统一异常**: EntityNotFoundError, DuplicateEntityError, ValidationError
- **HTTP状态码**: 200, 201, 204, 400, 404, 409
- **错误消息**: 友好的错误提示
