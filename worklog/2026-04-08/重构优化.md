## 工作概述

按照DDD领域驱动设计原则，对课程相关的数据结构进行了重新设计。将原有的年级、科目、章节三个实体拆分为独立的聚合根，通过外键引用建立关系，实现了数据结构的规范化管理。同时确保了历史数据的兼容性。

## 详细内容

### 1. 课程数据结构DDD设计

**设计理念:**
- 采用DDD领域驱动设计方法
- 识别聚合边界，定义聚合根
- 通过ID引用建立领域关系
- 封装领域行为和业务规则

**核心概念:**

#### Grade（年级聚合）
- **聚合根**: Grade
- **聚合内实体**: GradeSubject（年级-科目关联）
- **业务规则**: 
  - 年级是教育阶段的基本单位
  - 一个年级可以有多个科目
  - 删除年级时级联删除关联关系

#### Subject（科目聚合）
- **聚合根**: Subject
- **特点**: 独立的聚合根，可被多个年级使用
- **业务规则**:
  - 科目是学科分类
  - 一个科目可以被多个年级使用

#### Chapter（章节聚合）
- **聚合根**: Chapter
- **聚合内实体**: KnowledgePoint（知识点）
- **外键引用**: grade_id, subject_id
- **业务规则**:
  - 章节属于具体的年级和科目
  - 章节包含多个知识点

**聚合边界识别:**

```
Grade (聚合根)
  └─ 年级基本信息
  └─ GradeSubject (关联实体) - 该年级下的科目配置

Subject (聚合根)
  └─ 科目基本信息

Chapter (聚合根)
  ├─ 知识点（KnowledgePoint）- 实体
  └─ 章节依赖关系 - 值对象
```

**GradeRepository实现:**

```python
class GradeRepository:
    """年级仓储"""
    
    def __init__(self, data_file: str = "data/grades.json"):
        self.data_file = data_file
        self._grades: Dict[str, Grade] = {}
        self._load_data()
    
    def save(self, grade: Grade) -> Grade:
        """保存年级"""
        self._grades[grade.id] = grade
        self._save_data()
        return grade
    
    def find_by_id(self, grade_id: str) -> Optional[Grade]:
        """根据ID查找年级"""
        return self._grades.get(grade_id)
    
    def find_all(self) -> List[Grade]:
        """获取所有年级"""
        return list(self._grades.values())
    
    def delete(self, grade_id: str) -> None:
        """删除年级"""
        if grade_id in self._grades:
            del self._grades[grade_id]
            self._save_data()
```

**数据持久化:**
- 使用JSON文件存储（`data/grades.json`, `data/subjects.json`）
- 每次操作后自动保存
- 支持种子数据初始化

**GradeService业务逻辑:**

```python
class GradeService:
    """年级领域服务"""
    
    def create_grade(
        self, 
        name: str, 
        code: str, 
        level: GradeLevel,
        sort_order: int = 0,
        description: Optional[str] = None,
    ) -> Grade:
        """创建年级"""
        # 检查名称唯一性
        if self.grade_repo.find_by_name(name):
            raise DuplicateEntityError(f"Grade with name '{name}' already exists")
        
        # 检查代码唯一性
        if self.grade_repo.find_by_code(code):
            raise DuplicateEntityError(f"Grade with code '{code}' already exists")
        
        # 创建年级
        grade = Grade(
            id=f"G_{code}",
            name=name,
            code=code,
            level=level,
            subjects=[],
            sort_order=sort_order,
            description=description,
            status=Status.ACTIVE,
            created_at=datetime.now(),
            updated_at=datetime.now(),
        )
        return self.grade_repo.save(grade)
    
    def add_subject_to_grade(
        self, 
        grade_id: str, 
        subject_id: str,
        sort_order: int = 0,
    ) -> GradeSubject:
        """为年级添加科目"""
        # 验证年级存在
        grade = self.grade_repo.find_by_id(grade_id)
        if not grade:
            raise EntityNotFoundError(f"Grade '{grade_id}' not found")
        
        # 验证科目存在
        subject = self.subject_repo.find_by_id(subject_id)
        if not subject:
            raise EntityNotFoundError(f"Subject '{subject_id}' not found")
        
        # 检查是否已添加
        if any(gs.subject_id == subject_id for gs in grade.subjects):
            raise ValidationError(f"Subject '{subject_id}' already in grade '{grade_id}'")
        
        # 创建关联
        association = GradeSubject(
            id=f"GS_{grade_id}_{subject_id}",
            grade_id=grade_id,
            subject_id=subject_id,
            sort_order=sort_order,
            status=Status.ACTIVE,
            created_at=datetime.now(),
            updated_at=datetime.now(),
        )
        
        grade.subjects.append(association)
        self.grade_repo.save(grade)
        return association
```

```python
def migrate_chapter_references():
    """迁移章节的年级和科目引用"""
    # 年级名称到ID映射
    grade_mapping = {
        "初一": "G_C1",
        "初二": "G_C2",
        "初三": "G_C3",
        "高一": "G_S1",
        "高二": "G_S2",
        "高三": "G_S3",
    }
    
    # 科目名称到ID映射
    subject_mapping = {
        "数学": "S_MATH",
        "语文": "S_CHINESE",
        "英语": "S_ENGLISH",
        "物理": "S_PHYSICS",
        "化学": "S_CHEMISTRY",
    }
    
    # 迁移章节数据
    chapters = chapter_repo.find_all()
    for chapter in chapters:
        if chapter.grade in grade_mapping:
            chapter.grade_id = grade_mapping[chapter.grade]
        if chapter.subject in subject_mapping:
            chapter.subject_id = subject_mapping[chapter.subject]
        chapter_repo.save(chapter)
```

## 技术要点

### 1. DDD设计原则
- **聚合边界**: Grade、Subject、Chapter各自独立管理
- **领域行为**: 业务逻辑集中在领域模型中
- **外键引用**: 通过ID引用建立关系，降低耦合
- **数据一致性**: 通过领域服务保证数据完整性

### 2. 数据结构优化

| 对比项 | 原设计 | 新设计 |
|--------|--------|--------|
| 年级管理 | localStorage存储 | 独立数据表 |
| 科目管理 | localStorage存储 | 独立数据表 |
| 年级-科目关联 | 通过章节推断 | 独立关联表 |
| 章节关联 | 字符串字段 | 外键引用 |
| 删除年级 | 可能导致孤立章节 | 外键约束阻止 |
| 编辑年级/科目 | 需手动同步更新 | 自动更新关联 |

### 3. 扩展性设计
- 年级和科目可以独立扩展属性
- 章节可以灵活关联不同的年级和科目
- 新增年级-科目关联表支持灵活配置

### 4. 数据一致性保证
- **外键约束**: 保证引用完整性
- **唯一性约束**: 年级名称和代码唯一
- **级联删除**: 删除年级时级联删除关联关系
- **业务验证**: 删除前检查是否有关联章节
