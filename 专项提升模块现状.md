# 专项提升模块现状梳理

**更新时间**：2026-03-29
**模块**：improvement（专项提升）
**状态**：Agent 架构已搭建，核心功能可用，部分功能待完善

---

## ✅ 已完成

### 1. 业务逻辑层（Service）
- ✅ 诊断流程：初始诊断 + 澄清对话（最多 5 轮）
- ✅ 方案生成：LLM 生成 + 规则备选
- ✅ 教学讲解：调用 LLM 生成讲解内容 + 白板公式
- ✅ 小测评估：题目加载 + 答题评分 + 通过/失败判定
- ✅ 状态管理：会话状态转换（ANALYZING → CLARIFYING → DIAGNOSED → PLANNING → LEARNING → QUIZ → COMPLETED）

### 2. API 层（Workflow 模式）
- ✅ `POST /api/v1/improvement/start` - 创建会话
- ✅ `GET /api/v1/improvement/session/{id}` - 获取会话
- ✅ `POST /api/v1/improvement/session/{id}/clarify` - 提交澄清回答
- ✅ `POST /api/v1/improvement/session/{id}/plan` - 生成学习方案
- ✅ `POST /api/v1/improvement/session/{id}/step/{order}` - 开始学习步骤
- ✅ `POST /api/v1/improvement/session/{id}/step/{order}/complete` - 完成学习步骤
- ✅ `GET /api/v1/improvement/session/{id}/quiz` - 获取小测题目
- ✅ `POST /api/v1/improvement/session/{id}/quiz` - 提交小测答案

### 3. Agent 架构（新）
- ✅ Agent 主类：`ImprovementAgent`
- ✅ 5 个核心工具：diagnose, clarify, generate_plan, teach, evaluate_quiz
- ✅ Agent 系统提示：定义职责、工作流程、决策原则
- ✅ `/api/v1/improvement/agent/run` 端点（简化版本）
- ✅ 文档：CLAUDE.md 中的 Agent 架构章节

### 4. 数据持久化
- ✅ 会话数据存储：`ai-teacher-backend/data/students.json`
- ✅ 学生档案更新：掌握度、学习记录
- ✅ 小测结果保存

### 5. 前端集成
- ✅ 专项提升流程 UI（诊断 → 澄清 → 方案 → 学习 → 测试）
- ✅ 前端调用现有 workflow API

## 🔧 待完善

### 1. 诊断精度
**问题**：
- 澄清问题生成依赖 LLM，质量不稳定
- 诊断置信度计算过于简单（固定 0.55）

**改进方案**：
- [ ] 优化澄清问题模板，提高问题质量
- [ ] 基于学生回答历史计算置信度
- [ ] 添加诊断结果验证机制

**优先级**：P2（中）

### 2. 方案生成
**问题**：
- LLM 生成的方案步骤数不稳定
- 步骤间的知识点关联不明确

**改进方案**：
- [ ] 约束 LLM 输出步骤数（3-5 步）
- [ ] 验证步骤中的知识点是否有效
- [ ] 添加步骤间的依赖关系

**优先级**：P2（中）

### 3. 教学讲解
**问题**：
- 讲解内容长度不一致
- 白板公式生成不稳定

**改进方案**：
- [ ] 约束讲解内容长度
- [ ] 优化 KaTeX 公式生成
- [ ] 添加讲解内容质量评分

**优先级**：P2（中）

### 4. 小测评估
**问题**：
- 题目难度与学生水平不匹配
- 通过阈值固定（60%）

**改进方案**：
- [ ] 根据学生基础动态选择题目难度
- [ ] 根据学生表现动态调整通过阈值
- [ ] 添加题目难度标签

**优先级**：P2（中）

### 5. 错误处理
**问题**：
- LLM 调用失败时的降级方案不完善
- 会话异常状态恢复机制缺失

**改进方案**：
- [ ] 完善 fallback 响应
- [ ] 添加会话恢复机制
- [ ] 记录错误日志用于分析

**优先级**：P1（高）

### 6. 性能优化
**问题**：
- 每个步骤都调用 LLM，响应时间长
- 没有缓存机制

**改进方案**：
- [ ] 缓存讲解内容（相同知识点）
- [ ] 预生成常见问题的澄清问题
- [ ] 异步处理 LLM 调用

**优先级**：P3（低）

### 7. 监控和分析
**问题**：
- 缺少关键指标监控
- 无法分析学生学习效果

**改进方案**：
- [ ] 记录诊断准确率
- [ ] 记录学生通过率
- [ ] 记录平均学习时间
- [ ] 生成学习效果报告

**优先级**：P3（低）

## 🚀 快速开始

### 验证当前功能
```bash
# 启动后端
cd ai-teacher-backend
python3 run.py

# 启动前端
cd ai-teacher-frontend
npm run dev

# 访问 http://localhost:3000
# 登录 → 选择课程 → 点击"专项提升"
```

### 测试 Agent 端点
```bash
curl -X POST http://localhost:8008/api/v1/improvement/agent/run \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "course_id": "MATH_JUNIOR_01",
    "score_input": {
      "exam_name": "一元二次函数",
      "score": 60,
      "total_score": 100,
      "available_time": 30,
      "difficulty": "normal",
      "foundation": "average"
    }
  }'
```
