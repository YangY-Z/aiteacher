# AI虚拟教师系统 - 学生档案功能设计方案

**文档版本**: v1.1  
**编写日期**: 2026年3月31日  
**设计目标**: 建立学生学习档案数据模型，服务于AI教学决策和个性化教学

---

## 一、功能定位

学生档案是AI虚拟教师系统的**核心教学数据底座**，仅服务于内部教学决策，不面向用户展示：

| 目标 | 说明 |
|------|------|
| **学习者分类** | 根据学习数据自动识别学习者类型（新手/有基础/复习者/熟练者） |
| **教学策略推荐** | 为不同类型学习者匹配最优教学策略 |
| **薄弱点识别** | 分析错误模式，定位知识薄弱环节 |
| **回溯决策支持** | 为回溯决策提供数据支撑 |
| **Prompt优化** | 将画像数据注入教学Prompt，实现个性化教学 |

## 三、学习者分类逻辑

基于现有 `learner_profile.py` 中的分类规则：

| 学习者类型 | 判断条件 | 教学策略 |
|-----------|---------|---------|
| **novice（新手）** | 首次学习 + 前置知识掌握度 < 60% | 基础讲解 + 前置补救 + 多示例 + 慢节奏 |
| **intermediate（有基础者）** | 前置知识掌握度 ≥ 60% + 未学过当前知识点 | 知识讲解 + 引导发现 + 正常节奏 |
| **reviewer（复习者）** | 学过当前知识点 + 当前掌握度 < 70% | 快速复习 + 针对强化 + 重点讲解错误点 |
| **advanced（熟练者）** | 当前掌握度 ≥ 70% | 拓展深化 + 应用实践 + 快节奏 |

## 四、错误模式与教学调整

| 错误模式 | 触发条件 | 教学调整 |
|---------|---------|---------|
| concept_misunderstanding | 概念题错误率 > 30% | 增加概念辨析示例，使用对比讲解 |
| calculation_error | 计算题错误率 > 30% | 分解计算步骤，增加验算提示 |
| procedure_error | 步骤错误频率高 | 分步演示，提供步骤清单 |
| prerequisite_gap | 回溯到前置知识点 | 先补习前置知识，再继续学习 |
| careless_error | 粗心错误频率高 | 增加检查环节，提醒细心 |

### 5.2 核心方法说明

| 方法 | 用途 | 调用时机 |
|------|------|---------|
| `classify_learner` | 识别学习者类型 | 学习会话开始时 |
| `get_teaching_strategy` | 获取教学策略配置 | 构建Prompt时 |
| `get_error_patterns` | 获取错误模式列表 | 回溯决策时 |
| `get_weak_knowledge_points` | 获取薄弱知识点 | 选择回溯目标时 |
| `update_metrics` | 更新学习指标 | 学习会话结束时 |
| `add_error_pattern` | 记录错误模式 | 答题错误时 |

### 5.3 与现有服务的集成点

| 现有服务 | 集成方式 | 使用场景 |
|---------|---------|---------|
| `LearningService` | 调用 `classify_learner` | 开始学习时确定教学策略 |
| `BacktrackService` | 调用 `get_weak_knowledge_points` | 决定回溯目标 |
| `ChatService` | 调用 `get_teaching_strategy` | 构建教学Prompt |
| `TeachingModeService` | 调用 `get_error_patterns` | 选择教学模式 |
| `LearningService` 结束时 | 调用 `update_metrics` | 更新学习指标 |

### 7.2 动态教学要求生成

```python
def generate_teaching_requirements(profile: LearnerProfile, strategy: TeachingStrategy) -> str:
    """根据学习者画像生成教学要求"""
    
    requirements = []
    
    # 基于学习者类型
    if profile.learner_type == LearnerType.NOVICE:
        requirements.append("从基础概念开始讲解，使用通俗易懂的语言")
        requirements.append("每个知识点配合2-3个简单示例")
        requirements.append("讲解过程中多提问确认理解")
        
    elif profile.learner_type == LearnerType.INTERMEDIATE:
        requirements.append("可以直接讲解核心内容")
        requirements.append("引导学生发现规律和结论")
        
    elif profile.learner_type == LearnerType.REVIEWER:
        # 获取学生之前的错误点
        errors = get_recent_errors(profile.student_id, profile.course_id)
        requirements.append(f"重点复习学生之前出错的地方：{errors}")
        requirements.append("使用不同于之前的讲解方式")
        
    elif profile.learner_type == LearnerType.ADVANCED:
        requirements.append("简洁总结核心要点")
        requirements.append("增加拓展应用和综合题")
    
    # 基于错误模式
    dominant_error = get_dominant_error_pattern(profile.metrics.error_patterns)
    if dominant_error:
        if dominant_error.pattern_type == ErrorPatternType.CONCEPT_MISUNDERSTANDING:
            requirements.append("注意概念辨析，强调易混淆点")
        elif dominant_error.pattern_type == ErrorPatternType.CALCULATION_ERROR:
            requirements.append("计算步骤要详细展示，提醒检查")
        elif dominant_error.pattern_type == ErrorPatternType.PREREQUISITE_GAP:
            requirements.append("先复习相关前置知识")
    
    return "\n".join(requirements)
```

## 八、关键指标计算

| 指标 | 计算公式 | 数据来源 | 用途 |
|------|---------|---------|------|
| 前置知识掌握度 | 已掌握前置知识点数 / 前置知识点总数 | learning_records | 学习者分类 |
| 当前知识点掌握度 | 通过尝试次数 / 总尝试次数 | learning_records.attempts | 学习者分类 |
| 学习速度 | 学生平均答题时间 / 群体平均答题时间 | student_answers | 教学节奏调整 |
| 正确率 | 正确答题数 / 总答题数 | student_answers | 策略调整 |
| 回溯深度 | 平均每次学习回溯的知识点层级数 | learning_records | 薄弱点分析 |

## 十一、总结

学生档案功能作为AI虚拟教师系统的**核心教学数据底座**，不面向用户展示，而是服务于：

1. **学习者分类** - 自动识别学生类型，匹配教学策略
2. **错误分析** - 追踪错误模式，定位薄弱环节
3. **教学决策** - 为Prompt提供个性化参数
4. **回溯支持** - 为回溯决策提供数据支撑

该方案基于现有 `learner_profile` 模块扩展，无需大规模新增代码，重点在于**完善数据采集**和**集成到教学流程**。
