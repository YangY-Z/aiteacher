### 2.2 MVP阶段推荐方案

**主选方案: 智谱AI GLM-4**

推荐理由：
- 中文能力优秀，适合初中数学教学场景
- 国内直连，无需代理，响应稳定
- 价格适中，适合MVP阶段控制成本
- 数据存储在国内，符合教育行业合规要求
- 提供完善的API和SDK

**备选方案: 通义千问**

适用场景：
- 成本敏感时可作为降级方案
- 价格最低，适合大规模用户

### 2.3 智谱AI接入方案

```python
# 接入示例
from zhipuai import ZhipuAI

client = ZhipuAI(api_key="your_api_key")

def generate_teaching_response(
    system_prompt: str,
    user_message: str,
    temperature: float = 0.7,
    max_tokens: int = 1024
):
    response = client.chat.completions.create(
        model="glm-4",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_message}
        ],
        temperature=temperature,
        max_tokens=max_tokens
    )
    return response.choices[0].message.content
```

### 2.4 LLM调用成本估算

**场景假设**：
- 单个知识点讲解：平均1500 tokens（输入1000 + 输出500）
- 单次交互提问：平均500 tokens
- 单次回溯决策：平均800 tokens

**单个学生完成一次函数单元的成本估算**：

| 场景 | 调用次数 | 单次tokens | 总tokens | 费用估算 |
|------|---------|-----------|---------|---------|
| 知识点讲解 | 32次 | 1500 | 48,000 | 9.6元 |
| 交互提问 | 48次 | 500 | 24,000 | 2.4元 |
| 回溯决策 | 10次（预估） | 800 | 8,000 | 1.6元 |
| **合计** | - | - | **80,000** | **约13.6元/学生** |

**注**: 实际费用可能更低，因为部分学生不会触发回溯，且可使用缓存优化。

function CoordinateGraph({ formula, k, b }) {
  const chartRef = useRef(null)
  
  useEffect(() => {
    const chart = echarts.init(chartRef.current)
    
    // 生成直线数据点
    const data = []
    for (let x = -5; x <= 5; x += 0.5) {
      data.push([x, k * x + b])
    }
    
    chart.setOption({
      xAxis: { type: 'value', min: -5, max: 5 },
      yAxis: { type: 'value', min: -5, max: 5 },
      series: [{
        type: 'line',
        data: data,
        smooth: false,
        lineStyle: { color: '#4A90D9', width: 2 }
      }],
      grid: { containLabel: true }
    })
  }, [k, b])
  
  return <div ref={chartRef} style={{ width: '100%', height: '300px' }} />
}

## 六、数据库选型

### 6.1 PostgreSQL

**用途**: 存储核心业务数据

**数据表**:
- students（学生信息）
- courses（课程信息）
- knowledge_points（知识点）
- learning_records（学习记录）
- student_profiles（学习档案）

**选型理由**:
- 成熟稳定，ACID特性完善
- JSON支持好，可存储复杂数据结构
- 性能优秀，适合OLTP场景
- 开源免费，运维成本低

### 6.2 MongoDB

**用途**: 存储海量日志数据

**数据集合**:
- session_logs（会话日志）
- interaction_logs（交互日志）
- error_logs（错误日志）

**选型理由**:
- 文档型存储，适合日志数据
- 写入性能高，适合海量数据
- 灵活Schema，适合快速迭代

### 6.3 Redis

**用途**: 缓存和会话管理

**数据结构**:
- session:{session_id} - 会话状态（Hash）
- student:{student_id}:progress - 学习进度（String）
- kp:{kp_id}:template - 知识点模板缓存（String）
- rate_limit:{ip} - 请求限流（String）

**选型理由**:
- 高性能，响应时间<1ms
- 数据结构丰富
- 支持过期时间，适合会话管理

### 8.3 成本优化建议

1. **LLM调用优化**
    - 对常见讲解内容进行缓存
    - 使用更便宜的模型处理简单任务
    - 批量请求减少API调用次数

2. **语音服务优化**
    - 提供文字输入选项，减少语音使用
    - 对TTS结果进行缓存（相同内容不重复合成）

3. **服务器优化**
    - 使用Serverless架构降低闲置成本
    - 使用竞价实例降低计算成本
