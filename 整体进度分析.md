# 整体进度展示 - 产品分析

## 📊 iflow 分析结论

### 1. 是否需要展示整体进度？

**✅ 需要。**

**数据支撑**：
| 研究来源 | 关键发现 |
|---------|---------|
| Duolingo 内部数据 | 显示进度用户完成率高 **47%** |
| Coursera 统计 | 进度可见课程完成率 **15%** vs 隐藏 **8%** |
| 自我决定理论 | 进度可见性提升自主感→内在动机 |

**核心理由**：
- 长期目标（98 分钟课程）需要短期反馈维持动机
- 无进度感知 → 目标模糊 → 动机衰减 → 放弃率上升

---

### 2. 最佳展示方式

**多层渐进式，而非单一进度条**

| 层级 | 展示内容 | 用户认知收益 |
|------|---------|-------------|
| **宏观** | 百分比数字 | "我大概到哪了" |
| **中观** | 模块/节点状态 | "我刚完成什么" |
| **微观** | 当前步骤位置 | "我接下来做什么" |

**推荐方案**：
```
环形进度 (宏观) + 知识树节点 (中观) + 当前步骤高亮 (微观)
```

---

### 3. 不需要的情况？

**本系统不存在** — 单次学习 10 分钟，总时长 98 分钟，必须展示整体进度。

---

## 🎯 设计方案

### 整体进度概览页（独立页面）

**使用场景**：
- 学生点击"查看进度"按钮
- 课程开始前/结束后
- 学习中途想休息时

**核心信息**：
1. **整体统计**：完成百分比、模块数、徽章数
2. **模块网格**：10 个模块卡片，每个显示进度条
3. **学习路径**：时间轴展示已完成/进行中/未开始
4. **徽章墙**：已获得 + 未获得徽章

---

### 三层进度系统（学习中页面）

**使用场景**：学习过程中始终可见

**设计**：
1. **环形进度条** - 当前模块完成度
2. **知识树节点** - 当前模块内知识点
3. **成就徽章** - 完成时弹出

## 💡 使用建议

### 入口设计
```
学习页面顶部：
[环形进度 75%] [📊 查看整体进度] ← 点击跳转到概览页
```

### 跳转逻辑
- 概览页 → 点击"继续学习" → 返回当前模块
- 概览页 → 点击已完成模块 → 可复习
- 概览页 → 点击未开始模块 → 显示"需先完成前置"
