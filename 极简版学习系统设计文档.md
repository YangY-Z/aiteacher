# 极简版AI教师学习系统设计文档

**版本**: v1.0  
**日期**: 2026-03-24  
**定位**: 快速验证AI教学核心效果的MVP版本

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **极简交互** | 单列对话，无多余UI干扰 |
| **核心循环** | 只保留讲解→提问→反馈三步 |
| **功能聚焦** | 砍掉白板、学习地图、语音、诊断 |
| **保留价值** | 公式渲染 + 进度条是核心价值点 |
| **知识整合** | 32个知识点合并为10个核心模块 |

---

## 二、核心模块设计（10个模块）

```
│      └─ 原K12-K14: 定义、图象、性质                              │
│                                                                 │
│  M5: 一次函数定义                                                │
│      └─ 原K15-K16: 定义、与正比例函数的关系                      │
│                                                                 │
│  M6: 图象绘制                                                    │
│      └─ 原K17-K19: 描点法、图象特征、两点确定直线                │

│  M7: 斜率与截距                                                  │

│  M9: 解析式求解                                                  │
│      └─ 原K27-K29: 待定系数法、由图象求解析式、求交点            │
```

### 2.2 模块依赖关系

```
M1 ──→ M2 ──→ M3 ──→ M4 ──→ M5 ──→ M6 ──→ M7 ──→ M8 ──→ M9 ──→ M10
 │              │      │      │      │      │      │
 └──────────────┴──────┴──────┴──────┴──────┴──────┴──→ 线性依赖链
```

**简化策略**: 采用线性依赖，降低系统复杂度

### 2.3 模块详情表

| 模块ID | 模块名称 | 原知识点 | 核心学习目标 | 预估学习时长 |
|--------|---------|---------|-------------|-------------|

| M2 | 变量与函数 | K4, K5, K6, K7 | 理解变量、常量，掌握函数概念 | 8分钟 |

| M5 | 一次函数定义 | K15, K16 | 理解y=kx+b形式，与正比例函数的区别 | 8分钟 |
| M6 | 图象绘制 | K17, K18, K19 | 掌握描点法，理解"两点确定直线" | 10分钟 |
| M7 | 斜率与截距 | K20, K21, K22, K23, K24 | 掌握k和b的几何意义，理解增减性 | 12分钟 |
| M8 | 图象综合判断 | K25, K26 | 综合判断图象位置与方向 | 10分钟 |
| M9 | 解析式求解 | K27, K28, K29 | 掌握待定系数法，求交点 | 12分钟 |
| M10 | 实际应用 | K30, K31, K32 | 建模思想，与方程/不等式联系 | 15分钟 |

**总预估时长**: 约98分钟（1.5小时）

## 四、核心流程设计

### 4.1 三步循环流程

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│     ┌──────────┐                                            │
│     │  讲解    │ ← AI主动讲解当前模块内容                    │
│     │ Explain  │   - 引入概念                                               │
│     └────┬─────┘   - 给出定义                                │
│          │         - 展示公式                                │
│          ▼         - 举例说明                                │
│     ┌──────────┐                                            │
│     │  提问    │ ← AI提出检验性问题                          │
│     │ Question │   - 概念理解题                              │
│     └────┬─────┘   - 选择题/判断题                          │
│          │         - 填空题                                  │
│          ▼                                                │
│     ┌──────────┐                                            │
│     │  反馈    │ ← AI根据回答给予反馈                        │
│     │ Feedback │   - 正确：鼓励 + 下一模块                   │
│     └────┬─────┘   - 错误：解释 + 重新提问                   │
│          │                                                │
│          └────────────→ 循环直到模块完成                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 详细交互流程

```
开始学习
    │
    ▼
┌─────────────────────────────────────┐
│ 显示模块概览                         │
│ "今天学习：M5 一次函数定义"          │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 【讲解阶段】                         │
│                                     │
│ AI: "我们已经学了正比例函数 y=kx    │
│      现在来看一个更一般的形式..."    │
│                                     │
│ 展示公式: y = kx + b (k ≠ 0)        │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│ 【提问阶段】                         │
│                                     │
│ AI: "那么当b=0时，这个函数变成      │
│      什么了？"                       │
│                                     │
│ A. y = kx     B. y = b              │
│ C. 无法确定                           │
└──────────────────┬──────────────────┘
                   │
         ┌─────────┴─────────┐
         │                   │
    学生回答A            学生回答B/C
         │                   │
         ▼                   ▼
┌─────────────────┐  ┌─────────────────┐
│ 【反馈-正确】   │  │ 【反馈-错误】   │
│                 │  │                 │
│ "正确！当b=0时  │  │ "不太对，让我们 │
│  y=kx+b变成y=kx │  │  再想想..."     │
│  这就是正比例   │  │                 │
│  函数"          │  │ → 重新讲解      │
│                 │  │ → 换个问题      │
│ → 继续下一个    │  │                 │
│   问题或模块    │  └─────────────────┘
└─────────────────┘
```

### 4.3 模块完成判定

| 判定条件 | 规则 |
|---------|------|
| 通过 | 连续答对2道题 |
| 不通过 | 同一问题答错2次，触发"换个角度讲解" |
| 模块完成 | 通过后，显示总结 → 进入下一模块 |

### 6.3 数据模型

```typescript
// 模块定义
interface Module {
  id: string;           // "M1" - "M10"
  name: string;         // "坐标系基础"
  order: number;        // 1-10
  description: string;  // 模块描述
  formulas: string[];   // 核心公式列表
  questions: Question[]; // 内置问题库
}

// 学习进度
interface LearningProgress {
  studentId: string;
  currentModule: string;      // 当前模块ID
  completedModules: string[]; // 已完成模块
  status: 'learning' | 'questioning' | 'feedback';
  questionCount: number;      // 当前模块答题数
  correctCount: number;       // 当前模块正确数
}

// 消息类型
interface Message {
  id: string;
  role: 'ai' | 'student';
  content: string;
  formulas?: string[];   // 包含的公式
  options?: string[];    // 选择题选项
  timestamp: Date;
}
```

```typescript

Request:
```

## 七、核心Prompt设计

### 7.1 讲解Prompt

```
你是一位耐心的初中数学老师，正在辅导学生学习《一次函数》。

当前模块：{module_name}
模块描述：{module_description}
核心公式：{formulas}

学生情况：
- 这是学生第一次学习这个模块
- 已掌握的模块：{completed_modules}

讲解要求：
1. 用通俗的语言讲解概念，控制在100字以内
2. 重点公式用 $...$ 包裹，会被渲染成数学公式
3. 讲解结束后，提出一个检验性问题
4. 语气亲切，像朋友一样对话

请开始讲解：
```

### 7.2 反馈Prompt

```
学生回答：{student_answer}
正确答案：{correct_answer}
模块信息：{module_info}

反馈要求：
1. 如果正确：简短鼓励（20字内），然后进入下一问题或下一模块
2. 如果错误：解释错误原因（50字内），换个角度重新讲解

请给出反馈：
```
