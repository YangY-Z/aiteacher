# AI 虚拟教师系统 - 极简版设计方案

## 🎯 产品理念

**"少即是多" - 乔布斯**

砍掉一切不直接帮助学生理解知识点的功能，专注核心学习体验。

---

## ❌ 砍掉的功能

| 功能 | 砍掉理由 |
|------|---------|
| 白板组件 | 分散注意力，公式在对话中渲染就够了 |
| 学习地图可视化 | 进度条就够了，学生不需要看知识图谱 |
| 课前诊断测试 | 增加入口摩擦，直接开始学习 |
| 语音输入输出 | 数学学习打字更快，语音是噱头 |
| 6 种教学模式 | 学生感知不到差异，统一成一种 |
| 跳过知识点功能 | 这是逃避的后门，应该证明掌握才能跳过 |
| 复杂的回溯决策 | 预设依赖关系就够了，不需要 LLM 推断 |
| 监控看板 | MVP 阶段没用，给运营看的不是给学生 |
| 分离的聊天和白板 | 合并成单列对话流 |

---

## ✅ 保留的核心

1. **单列对话式布局** - 专注对话，无干扰
2. **三步学习循环** - 讲解 → 提问 → 反馈
3. **公式渲染** - KaTeX 支持数学公式
4. **进度条** - 简单清晰的进度展示
5. **选择题交互** - 即时反馈
6. **流式输出** - 打字机效果，实时响应

## 📐 极简版架构

### 界面布局

```
┌────────────────────────────────────────────┐
│  一次函数 · 进度 3/10                       │
│  ████████████░░░░░░░░░░ 30%               │
├────────────────────────────────────────────┤
│                                            │
│  🤖 一次函数的标准形式是 y = kx + b        │
│     其中 k 是斜率，b 是截距                  │
│                                            │
│  🤖 下面哪个是一次函数？                    │
│     A. y = 2x + 3                          │
│     B. y = x²                              │
│     C. y = 2/x                             │
│                                            │
│  👤 我选 A                                  │
│                                            │
│  🤖 ✅ 正确！y = 2x + 3 中 x 的次数是 1...   │
│                                            │
├────────────────────────────────────────────┤
│  [输入问题或答案...]            [发送]     │
└────────────────────────────────────────────┘
```

### 学习流程

```
开始
  ↓
讲解 (1-2 分钟，流式输出)
  ↓
提问 (1 道选择题)
  ↓
学生回答
  ↓
┌─────────────┐
│ 答对了？    │
└─────────────┘
   │        │
   是       否
   │        │
   ↓        ↓
 下一个   补讲 30 秒
 知识点     ↓
          再问 1 题
            ↓
          (循环)
```

**删除/弃用**:

- `components/layout/LearningMap.tsx` - 学习地图
- `components/diagnostic/DiagnosticTest.tsx` - 诊断测试

**简化**:
- `Learning.tsx` (450 行) → `MinimalLearning.tsx` (200 行)
- 状态管理从 8 个 reducer → 4 个 useState
- API 调用从 6 个端点 → 2 个端点

### 后端改动

**简化 API**:
```
原 API (12 个端点):
  /api/v1/auth/login
  /api/v1/auth/register
  /api/v1/students/profile
  /api/v1/courses
  /api/v1/courses/{id}
  /api/v1/diagnostic/start
  /api/v1/diagnostic/submit
  /api/v1/learning/start
  /api/v1/learning/chat
  /api/v1/learning/response
  /api/v1/learning/assess
  /api/v1/retention/...

极简 API (5 个端点):
  /api/v1/learning/start      # 开始学习
  /api/v1/learning/chat       # 发送消息 (流式)
  /api/v1/learning/assess     # 提交答案
  /api/v1/progress            # 获取进度
  /api/v1/auth/login          # 登录 (保留)
```

**简化数据模型**:
```python
# 原来
LearningSession {
  id, student_id, course_id, knowledge_point_id,
  phase, attempts, whiteboard_content, chat_history,
  assessment_results, backtrack_history, ...
}

# 极简版
LearningSession {
  id, student_id, current_kp_id,
  messages: [], progress: 0-100
}
```

## 🎨 设计原则

### 1. 专注
- 一页只做一件事：学习
- 无侧边栏、无抽屉、无弹窗
- 所有操作在对话中完成

### 2. 即时反馈
- 答题后立即显示对错
- 进度条实时更新
- 打字机效果让学生知道 AI 在思考

### 3. 减少决策
- 不让学生选择教学模式
- 不让学生选择复习内容
- 系统自动推荐下一步

### 4. 成就感
- 每学完一个模块有明确提示
- 进度条可视化增长
- 简单但有力的鼓励语
