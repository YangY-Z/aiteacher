# 流式输出方案实现总结

## 一、需求背景

AI虚拟教师系统需要实现：
- 白板内容先展示（知识点要点、公式）
- 消息流式展示（引入、定义、示例、总结、提问）
- 单次LLM调用
- 用户无感知JSON结构

---

## 二、方案演进

### 方案对比

| 方案 | LLM输出格式 | 流式支持 | 实现复杂度 | 最终选择 |
|------|------------|---------|-----------|---------|
| 完整JSON | JSON对象 | ❌ 需等待闭合 | 低 | - |
| 两阶段调用 | 两次JSON | ✅ | 中 | - |
| Markdown格式 | Markdown | ✅ 需解析 | 中 | - |
| 标签格式 | `[TAG]...[/TAG]` | ✅ | 中 | - |
| **JSONL格式** | 每行一个JSON | ✅ 直接解析 | 低 | ✅ |

### 最终方案：JSONL + 增量事件

**LLM输出示例：**
```jsonl
{"type":"wb_title","content":"函数解析式"}
{"type":"wb_points","content":"用数学式子表示函数关系"}
{"type":"wb_formulas","content":"y = kx + b"}
{"type":"msg_intro","content":"函数关系可以用数学式子来表达..."}
{"type":"complete","next_action":"wait_for_student"}
```

## 三、典型问题与解决方案

### 问题1：LaTeX反斜杠转义问题

**现象：**
```
k = rac{y}{x} (x≠0)  // \frac 变成了 rac
```

**原因：**
JSON中 `\f` 是换页符(form feed)，`\frac` 被解释为 `换页符 + rac`

**解决方案：**
后端解析JSONL时，自动修复反斜杠转义：

```python
import re

# 将单反斜杠转为双反斜杠（但保护已正确转义的）
line_fixed = re.sub(r'\\(?!["\\/bfnrtu])', r'\\\\', line)
data = json.loads(line_fixed)
```

**正则解释：**
- `\\(?!["\\/bfnrtu])` 匹配后面不是有效JSON转义字符的反斜杠
- `\frac` → `\\frac`，`\pi` → `\\pi`
- `\n`、`\t` 等保持不变

### 问题2：React StrictMode双重调用

**现象：**
- 消息重复显示两条
- 白板内容重复
- 点击一次选项，显示两次选择

**原因：**
React 18 StrictMode 在开发模式下会执行两次副作用和回调，导致：
- `Date.now()` 两次调用产生不同ID → 去重失效
- 状态更新函数执行两次 → 内容重复

**解决方案：**

1. **消息去重（内容+时间判断）：**
```typescript
addMessage: (message) => set((state) => {
  // 检查ID重复
  if (state.messages.some(m => m.id === message.id)) {
    return state;
  }
  // 检查最近消息是否内容相同且时间间隔小于1秒
  const lastMessage = state.messages[state.messages.length - 1];
  if (lastMessage && 
      lastMessage.role === message.role && 
      lastMessage.content === message.content &&
      Date.now() - lastMessage.timestamp.getTime() < 1000) {
    return state;
  }
  return { messages: [...state.messages, message] };
}),
```

2. **白板增量更新去重：**
```typescript
addWhiteboardPoint: (point) => set((state) => {
  // 如果要点已存在，不重复添加
  if (state.currentWhiteboard.key_points.includes(point)) return state;
  return {
    currentWhiteboard: {
      ...state.currentWhiteboard,
      key_points: [...state.currentWhiteboard.key_points, point],
    },
  };
}),
```

### 问题3：白板重复渲染

**现象：**
白板内容展示两份相同的内容

**原因：**
白板同时渲染两个数据源：
- `currentWhiteboard`：增量更新的实时内容
- `whiteboardBlocks`：已提交的内容块

`commitWhiteboard()` 后没有清空 `currentWhiteboard`

**解决方案：**
```typescript
commitWhiteboard: () => set((state) => {
  // ... 提交逻辑
  
  return { 
    whiteboardBlocks: [...state.whiteboardBlocks, newBlock],
    // 关键：提交后清空 currentWhiteboard
    currentWhiteboard: initialWhiteboardState,
  };
}),
```

### 问题4：白板内容不实时显示

**现象：**
后端已发送 `wb_title` 事件，但白板没有立即显示

**原因：**
白板组件只渲染 `whiteboardBlocks`，而增量更新写入的是 `currentWhiteboard`

**解决方案：**
白板组件同时渲染两个数据源：

```tsx
// 渲染已提交的白板块
{whiteboardBlocks.map((block, index) => renderBlock(block, index))}

// 渲染当前正在增量更新的内容（实时显示）
{currentWhiteboard.title && (
  <div className="whiteboard-block streaming">
    {/* 渲染 currentWhiteboard 内容 */}
  </div>
)}
```

---

### 问题5：Markdown格式不渲染

**现象：**
消息显示 `**提问**：你能说出区别吗？` 而不是粗体

**解决方案：**
添加简单Markdown渲染器：

```typescript
const renderMarkdown = (text: string) => {
  const parts = text.split(/(\*\*[^*]+\*\*)/g);
  return parts.map((part, index) => {
    if (part.startsWith('**') && part.endsWith('**')) {
      return <strong key={index}>{part.slice(2, -2)}</strong>;
    }
    return part;
  });
};
```

## 四、数据流架构

```
┌─────────────┐    流式字符     ┌─────────────┐    增量事件     ┌─────────────┐
│   LLM输出    │ ────────────→ │   后端解析   │ ────────────→ │   前端渲染   │
│   JSONL     │               │   JSONL→事件 │               │   React状态  │
└─────────────┘               └─────────────┘               └─────────────┘

事件类型：
- wb_title    → 白板标题（实时更新）
- wb_points   → 白板要点（增量追加）
- wb_formulas → 白板公式（增量追加）
- msg_intro   → 引入消息
- msg_def     → 定义消息
- complete    → 提交白板，完成
```

## 六、最佳实践

1. **流式输出优先选择JSONL格式** - 每行独立解析，支持真正的流式
2. **React状态更新需考虑StrictMode** - 添加内容去重和时间判断
3. **LaTeX公式注意转义** - 后端统一处理反斜杠转义
4. **增量更新与提交分离** - 实时显示用临时状态，提交后清空
5. **SSE事件类型化** - 不同事件类型对应不同处理逻辑
