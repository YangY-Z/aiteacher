# 📊 学生进度感知系统 - 设计方案

## 🎯 问题分析

**原问题**：学生无法清晰感知当前学习进度

**iflow 分析结果**：
- 简单的进度条信息维度单一
- 学生需要多层次的进度反馈
- 游戏化元素能提升学习动力

---

## ✅ 解决方案：多维度进度展示

根据 iflow 的分析，我设计了**三层进度感知系统**：

### 1️⃣ 环形进度条（宏观进度）
- **位置**：顶部左侧
- **展示内容**：当前模块完成百分比
- **特点**：
  - 60x60px 圆环，占用空间小
  - 渐变色（蓝→紫）视觉吸引力强
  - 中心显示百分比数字
  - 平滑动画过渡

### 2️⃣ 知识树进度（微观进度）
- **位置**：顶部底部
- **展示内容**：5-10 个知识点节点
- **特点**：
  - 节点状态：已完成✓（绿色）、进行中●（蓝色脉动）、未开始○（灰色）
  - 每个节点显示知识点名称
  - 清晰展示学习路径
  - 当前节点脉动动画，吸引注意

### 3️⃣ 成就徽章（激励进度）
- **位置**：顶部下方（获得时显示）
- **展示内容**：已获得的徽章
- **特点**：
  - 游戏化设计，增强成就感
  - 滑动出现动画
  - 可收集性，激发学习动力
  - 示例徽章：快速学习者、连续学习 7 天、首次满分等

## 🎨 设计亮点

### 1. 多层次进度感知
- **宏观**：百分比数字（0-100%）
- **中观**：环形进度条（视觉化）
- **微观**：知识树节点（具体知识点）
- **激励**：徽章系统（成就感）

### 2. 视觉反馈
- **颜色编码**：
  - 蓝色 #007aff - 进行中
  - 绿色 #34c759 - 已完成
  - 灰色 #f0f0f0 - 未开始
  - 橙色 #ff9500 - 成就徽章

- **动画效果**：
  - 环形进度平滑过渡（0.5s）
  - 当前节点脉动（1.5s 循环）
  - 徽章滑入动画（0.4s）
  - 徽章弹出动画（0.3s）

### 3. 空间优化
- **紧凑布局**：所有进度信息在顶部 150px 内
- **响应式**：小屏幕自动隐藏节点名称
- **固定定位**：滚动时始终可见

## 💡 实现细节

### 环形进度条计算
```javascript
const radius = 24;
const circumference = 2 * Math.PI * radius; // 150.8
const progress = 75; // 75%
const offset = circumference - (progress / 100) * circumference;
// strokeDashoffset = 37.7
```

### 知识树状态管理
```javascript
knowledgePoints.forEach((kp, idx) => {
  if (idx < currentStep) kp.status = 'completed';   // 绿色✓
  else if (idx === currentStep) kp.status = 'current'; // 蓝色●
  else kp.status = 'locked'; // 灰色○
});
```

### 徽章触发逻辑
```javascript
if (currentStep === totalSteps && !badges.includes('快速学习者')) {
  badges.push('快速学习者');
  showBadge('快速学习者');
}
```
